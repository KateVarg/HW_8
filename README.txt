2. Почему для хранения товаров в корзине используется словарь, а не список?
Если бы для хранения товаров использовался список, то у каждому параметру товара присваивался бы индекс, что не удобно при работе с данными параметрами. При использовании словаря, все параметры хранятся в виде ключ: значение, т.е. у каждого параметра есть название и его значение.

3. Зачем нужен hash в классе Product? Изучите этот метод и объясните, почему он нужен.
hash нужен для увеличения скорости обработки данных. Каждой паре ключ: значение присваивается hash. И при дальнейшей работе с данными происходит быстрый просмотр hash значений в словаре.
